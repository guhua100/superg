git的简单实用命令记录

1、git 更新子模块commit id

git是一个分布式的代码管理软件，git在管理子模块项目时，初学者经常会不会同步更新子模块的更新，原因就是在于git clone主项目时，是根据commit id来更新submodule的，默认commit不会更新commit id

  • 解决方法，git add的时候加 -u选项，表示增加、修改、删除、更新的地方都加入到缓存区
git add -u
git commit -a -m ""

2、git使用多个git库时配置文件配置

git 默认的id文件是id_rsa，当不同的git库使用时会产生冲突

  • 解决方法： 在~/.ssh/config里增加配置：
Host github.com api.github.com
IdentityFile /path/to/github.com.id_rsa

其它机器git库以此类推 测试

3、git强制覆盖：

    git fetch --all
    git reset --hard origin/master
    git pull
  • git强制覆盖本地命令（单条执行）：

git fetch --all && git reset --hard origin/master && git pull
